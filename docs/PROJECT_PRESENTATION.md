# 项目讲解指南

这份指南用于准备 3 到 8 分钟的项目介绍。

## 30 秒版本

Campus Life Platform 是我做的一个高校校园生活综合服务平台，包含学生端、管理后台和两套 Node.js API。学生端覆盖校园动态、二手集市、悬赏任务、订单模拟支付和聊天通知；管理后台覆盖用户、学校、动态、商品、悬赏和公告管理。项目使用 React、Ant Design、Redux Toolkit、Node.js、Express、MySQL、JWT 和 WebSocket。

## 3 分钟版本

1. **背景**：校园信息、交易、任务互助和学校管理入口通常比较分散，所以我把它抽象成一个校园生活平台。
2. **架构**：前端分为学生端和管理端，后端分为用户 API 和管理 API，数据库使用 MySQL。
3. **学生端能力**：动态发布与评论、校园集市、悬赏任务、订单模拟支付、聊天和通知。
4. **管理端能力**：平台管理员和学校管理员可以管理用户、学校、动态、商品、悬赏任务和公告。
5. **设计沉淀**：项目中保留了用例规约、通信图、顺序图、类图、数据库设计和开发记录，能说明从需求到实现的完整过程。

## 项目亮点表达

- **不是单页面 demo**：包含学生端、管理端、用户 API、管理 API 和数据库脚本。
- **业务闭环完整**：集市有商品发布、详情、订单和模拟支付；悬赏有发布、接单、进度和确认。
- **有权限边界**：用户、平台管理员、学校管理员职责不同。
- **有实时交互**：WebSocket 支撑聊天和通知。
- **有设计文档**：能展示需求分析和系统建模能力。

## 常见追问准备

**为什么拆成两套后端？**

用户端 API 和管理端 API 的权限边界不同。拆开后，学生端业务和后台审核管理可以分别演进，也方便说明系统边界。

**为什么保留设计草稿？**

保留设计草稿可以说明项目不是只做页面，而是经历了需求分析、用例设计、数据库建模和实现迭代。

**如果继续优化，你会先做什么？**

我会先补启动脚本、截图、核心测试和接口文档，然后再处理日志规范和部署脚本。
