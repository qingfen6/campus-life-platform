# 文档索引

这里整理的是面向求职展示的阅读入口。根目录保持项目代码为主，设计资料集中到 `docs/design/`，早期开发记录和实验草稿归档到 `docs/archive/`。

## 推荐阅读顺序

1. [系统架构说明](ARCHITECTURE.md)：了解项目的多端结构、数据流和核心模块。
2. [接口概览](API_OVERVIEW.md)：快速查看用户端 API 与管理端 API 的边界。
3. [设计资料总览](design/README.md)：查看用例、类图、关系模型、数据库设计和实现记录。
4. [面试讲解指南](INTERVIEW_GUIDE.md)：准备项目介绍、亮点说明和可改进点。
5. [截图与视觉资料](SCREENSHOTS.md)：查看已有视觉素材和后续截图补齐清单。
