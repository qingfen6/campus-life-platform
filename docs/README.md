# 文档索引

这里整理的是面向求职展示的阅读入口。原始设计文件、实验草稿和开发记录仍保留在项目根目录，方便说明项目从需求分析到实现落地的全过程。

## 推荐阅读顺序

1. [系统架构说明](ARCHITECTURE.md)：了解项目的多端结构、数据流和核心模块。
2. [接口概览](API_OVERVIEW.md)：快速查看用户端 API 与管理端 API 的边界。
3. [面试讲解指南](INTERVIEW_GUIDE.md)：准备项目介绍、亮点说明和可改进点。
